01

Context & Objective

Medical clinics and support centers dedicate massive operational hours to manual scheduling: booking slots, rescheduling visits, processing cancellation requests, and sending reminders.

The objective was to deploy a conversational booking assistant on WhatsApp. It needed to extract intent contextually, verify doctor availability in real-time, write details to scheduling backends, and maintain conversation state cleanly over multi-turn dialogues.

02

Technical Design

  • ⚡
    State & Session Management WhatsApp messaging triggers single stateless webhooks. To coordinate booking parameters across several turns (e.g. date -> doctor -> time slot), we implemented a Redis caching layer to preserve transient booking contexts indexed by user phone numbers.
  • 📊
    Entity Extraction & Validation Patients describe timing preferences dynamically (e.g., "next Tuesday after 3 PM"). We structured LLM parsing blocks to translate loose conversations into strict JSON schemas, matching slot constraints before writing entries.

03

Workflow Architecture

The setup relies on a clean, message-driven pipeline built on n8n workflows:

1 Meta Webhook Trigger Pushes patient WhatsApp text to n8n webhook.
2 Session Context Check Pulls state cache from Redis to retrieve active booking properties.
3 LLM Entity Extractor Extracts doctor specialties and preferred dates into structured slots.
4 Schedule Verification & Sync Queries slot sheets, updates bookings, clears Redis, and responds to WhatsApp.
