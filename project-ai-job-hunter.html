Autonomous AI Job Hunter Agent

01

Context & Objective

Managing the modern job search workflow involves a repetitive bottleneck: crawling listings daily, evaluating alignment with one's skills, modifying the resume to match core keywords without fabricating facts, writing cover letters, and tracking responses across sheets.

The objective was to create a fully autonomous closed-loop agent that does the heavy lifting: crawling jobs continuously, running local filtering to save compute costs, tailoring ATS-friendly resumes, and updating spreadsheets and emails in the background.

02

Challenges & Design

  • ⚠️
    API Computations & Cost Querying commercial LLMs for hundreds of job postings is financially unfeasible. We introduced a local Qwen-2.5:3B classifier node running in Ollama to pre-filter and reject low-match positions before sending high-fit listings to Gemini-2.5-Flash.
  • 🔒
    Anti-Hallucination Safe Locks Tailored resumes must remain 100% truthful. By structuring Gemini outputs to select and align *only* valid elements matching the input profile schema, we successfully blocked hallucinated details.

03

The Pipeline Architecture

The system uses state-managed loops in LangGraph to coordinate the workflow stages:

1 Crawl Stage Apify scrapes LinkedIn listings into an intake queue.
2 Cost-Saving Filter Local Qwen-2.5-3B filters out misaligned matches (e.g. Senior Roles, different tech).
3 Resume Tailoring Engine Gemini dynamically updates JSON schema representing candidate resume highlights.
4 Publishing Node Converts HTML templates to PDF via Gutenberg, saves to Google Drive, and notifies.
